4. Testes de Acessibilidade (Como Testar)

Aprenda a testar acessibilidade manualmente e com ferramentas automatizadas.

Testes Manuais

Navegação Apenas com Teclado

Objetivo: Verificar se todos os elementos interativos são acessíveis via teclado.

Passo a Passo:
  1. Tab: Avança para o próximo elemento focável
  2. Shift + Tab: Volta para o elemento anterior
  3. Enter / Espaço: Ativa botões e links
  4. Esc: Fecha modais, dropdowns e menus
  5. Setas: Navega em listas, menus e componentes customizados
Checklist:
  • Todos os elementos interativos são focáveis
  • Ordem de tabulação segue a lógica visual
  • Foco visível em todos os elementos
  • Não há "armadilhas de foco" (foco preso em modais)
  • Teclas de atalho funcionam (Enter para enviar, Esc para cancelar)

Uso de Leitores de Tela

Objetivo: Verificar como leitores de tela anunciam o conteúdo e a navegação.

Leitores de Tela Gratuitos:
  • NVDA: Windows (gratuito) - Download NVDA
  • JAWS: Windows (pago, trial disponível)
  • VoiceOver: Mac/iOS (integrado) - Cmd+F5 para ativar
  • TalkBack: Android (integrado)
  • Narrator: Windows (integrado) - Win+Ctrl+Enter
Teste com NVDA (Windows):
  1. Baixe e instale o NVDA
  2. Inicie o NVDA (Ctrl+Alt+N)
  3. Navegue pela página usando setas, Tab, e comandos NVDA
  4. Verifique se todos os elementos são anunciados corretamente
  5. Teste formulários, botões, links e navegação
O que verificar:
  • Textos alternativos de imagens são lidos
  • Labels de inputs são anunciados
  • Botões com aria-label são descritos corretamente
  • Estrutura de cabeçalhos (H1, H2, etc.) é navegável
  • Landmarks ARIA (nav, main, banner) são identificados

Teste de Foco Visível

Objetivo: Verificar se o foco é claramente visível durante navegação por teclado.

Como testar:
  1. Navegue pela página usando apenas a tecla Tab
  2. Observe visualmente onde o foco está em cada elemento
  3. Verifique se o indicador de foco é claro e visível
  4. Teste em diferentes navegadores (Chrome, Firefox, Safari, Edge)
Verificar:
  • Outline ou indicador de foco visível (mínimo 2px de espessura)
  • Contraste adequado entre foco e fundo
  • Foco não é removido com outline: none sem alternativa
  • :focus-visible usado para mostrar foco apenas em navegação por teclado

Zoom até 200% sem Quebra

Objetivo: Verificar se o layout permanece funcional com zoom até 200%.

Como testar:
  1. Chrome/Firefox: Ctrl+ (Windows) ou Cmd+ (Mac) para aumentar zoom
  2. Safari: Cmd+ para aumentar zoom
  3. Aumente o zoom gradualmente até 200%
  4. Verifique se o layout permanece funcional e legível
  5. Teste também no dispositivo móvel (Configurações → Tela → Tamanho da fonte)
Verificar:
  • Texto permanece legível sem rolagem horizontal
  • Elementos interativos não ficam sobrepostos
  • Navegação continua funcional
  • Layout não quebra ou vaza da viewport
  • Imagens escalam corretamente

Testes Automatizados

Importante: Testes automatizados detectam apenas ~30-40% dos problemas de acessibilidade. Use-os como complemento aos testes manuais, não como substituto.

Lighthouse (Acessibilidade)

Ferramenta: Integrada no Chrome DevTools, avaliador completo de performance, acessibilidade e SEO.

Como usar:
  1. Abra a página no Chrome
  2. Pressione F12 para abrir DevTools
  3. Vá para a aba "Lighthouse"
  4. Marque "Acessibilidade" (pode marcar outras também)
  5. Clique em "Generate report"
  6. Analise os resultados e correções sugeridas
O que avalia:
  • Contraste de cores
  • Textos alternativos de imagens
  • Labels em formulários
  • Uso correto de ARIA
  • Nomes acessíveis de elementos
  • Ordem de cabeçalhos
Limitações:
  • Não detecta problemas de lógica ou contexto
  • Pode dar falsos positivos/negativos
  • Não substitui testes manuais com leitores de tela

axe DevTools

Ferramenta: Extensão de navegador baseada na biblioteca axe-core (padrão de mercado).

Como usar:
  1. Instale a extensão axe DevTools
  2. Abra a página no navegador
  3. Abra DevTools (F12)
  4. Vá para a aba "axe DevTools"
  5. Clique em "Scan" para analisar a página
  6. Revise os problemas encontrados com explicações detalhadas
O que avalia:
  • Conformidade WCAG 2.1/2.2 (níveis A, AA, AAA)
  • Melhores práticas de ARIA
  • Estrutura semântica HTML
  • Problemas de contraste e cor
Limitações:
  • Analisa apenas o HTML estático carregado
  • Pode não detectar problemas de conteúdo dinâmico
  • Requer interpretação humana dos resultados

WAVE (Web Accessibility Evaluation Tool)

Ferramenta: Extensão de navegador ou ferramenta online para avaliação visual de acessibilidade.

Como usar:
  1. Instale a extensão WAVE ou use WAVE Online
  2. Abra a página no navegador
  3. Clique no ícone WAVE na barra de extensões
  4. Visualize os ícones de erro/aviso sobrepostos na página
  5. Clique nos ícones para ver detalhes e correções
O que avalia:
  • Estrutura semântica (cabeçalhos, listas, landmarks)
  • Textos alternativos
  • Contraste de cores
  • Formulários e labels
  • ARIA e roles
Limitações:
  • Pode mostrar falsos positivos em componentes JavaScript complexos
  • Visual pode ser poluído em páginas complexas

Pa11y

Ferramenta: Ferramenta de linha de comando para testes automatizados em CI/CD.

Como usar:
  1. Instale via npm: npm install -g pa11y
  2. Execute: pa11y https://example.com
  3. Para relatório HTML: pa11y --reporter html https://example.com > report.html
  4. Integre em CI/CD para validação contínua
O que avalia:
  • Usa axe-core ou HTML CodeSniffer como engine
  • Conformidade WCAG
  • Ideal para automação e integração contínua
Limitações:
  • Requer conhecimento de linha de comando
  • Configuração mais complexa
