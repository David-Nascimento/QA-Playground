3. Playgrounds Interativos

Explore problemas comuns de acessibilidade e suas soluções de forma prática.

Explore problemas comuns de acessibilidade ativando/desativando erros. Compare a versão com problema com a versão corrigida e aprenda na prática.

Playground 1: Botão sem aria-label

Versão com Problema

Problema: Botão apenas com ícone, sem texto ou aria-label. Usuários de leitores de tela não conseguem entender a função.

Leitores de tela anunciarão: "botão" (sem contexto)

<button>[Ícone]</button>
<!-- Sem aria-label ou texto -->

Versão Corrigida

Solução: Adicionar aria-label descritivo ou texto acessível.

Leitores de tela anunciarão: "Ver notificações, botão"

<button aria-label="Ver notificações">[Ícone]</button>

<!-- Ou com texto visível: -->
<button>[Ícone] <span class="sr-only">Ver notificações</span></button>

Por que funciona: O atributo aria-label fornece um nome acessível para tecnologias assistivas, mesmo quando o botão contém apenas um ícone visual.

Playground 2: Contraste Insuficiente

Versão com Problema

Problema: Contraste de cores abaixo do mínimo WCAG AA (4.5:1 para texto normal, 3:1 para texto grande).

Este texto tem contraste insuficiente (cinza claro sobre cinza)

Rácio de contraste: ~2.1:1 (abaixo do mínimo de 4.5:1)

.texto {
  color: #999999; /* Cinza claro - baixo contraste */
  background: #f0f0f0;
}

Versão Corrigida

Solução: Garantir contraste mínimo de 4.5:1 (texto normal) ou 3:1 (texto ≥18pt ou ≥14pt bold).

Este texto tem contraste adequado (preto sobre branco)

Rácio de contraste: ~12.6:1 (acima do mínimo de 4.5:1)

.texto {
  color: #000000; /* Preto - alto contraste */
  background: #ffffff;
  /* Contraste: 21:1 (muito acima do mínimo) */
}

Ferramentas para verificar: Use o WebAIM Contrast Checker ou o DevTools do navegador para verificar rácios de contraste.

Playground 3: Falta de Foco Visível

Versão com Problema

Problema: Foco removido com outline: none sem alternativa visual. Usuários de teclado não sabem onde estão.

Pressione Tab - nenhum indicador de foco visível

button:focus {
  outline: none; /* Remove foco sem alternativa - PROBLEMA */
}

Versão Corrigida

Solução: Manter outline padrão ou criar indicador de foco visível alternativo.

Pressione Tab - indicador de foco visível (outline + box-shadow)

button:focus-visible {
  outline: 3px solid var(--accent);
  outline-offset: 2px;
  box-shadow: 0 0 0 2px rgba(122, 168, 116, 0.2);
}

button:focus:not(:focus-visible) {
  outline: none; /* Remove foco apenas para mouse */
}

Por que funciona: O :focus-visible mostra foco apenas para navegação por teclado, mantendo a experiência visual limpa para usuários de mouse, mas acessível para usuários de teclado.

Playground 4: Ordem de Tabulação Incorreta

Versão com Problema

Problema: Ordem de tabulação não segue a ordem visual/lógica usando tabindex incorretamente.

Pressione Tab - ordem: Email → Telefone → Nome → Enviar (confuso!)

<input type="text" tabindex="3" /> <!-- Nome -->
<input type="email" tabindex="1" /> <!-- Email -->
<input type="tel" tabindex="2" /> <!-- Telefone -->
<!-- PROBLEMA: Ordem não segue a lógica visual -->

Versão Corrigida

Solução: Deixar ordem natural do DOM ou usar tabindex sequencial respeitando ordem visual.

Pressione Tab - ordem: Nome → Email → Telefone → Enviar (lógica!)

<input type="text" /> <!-- Nome - sem tabindex -->
<input type="email" /> <!-- Email - ordem natural -->
<input type="tel" /> <!-- Telefone - segue DOM -->
<!-- CORRETO: Ordem segue a estrutura HTML -->

Regra de ouro: Evite tabindex positivo. Deixe a ordem natural do DOM. Use tabindex="0" para tornar elementos não-focáveis focáveis (como divs interativas). Use tabindex="-1" para remover da ordem (geralmente para programação de foco via JavaScript).

Playground 5: Inputs sem Label

Versão com Problema

Problema: Input sem label associado. Leitores de tela não sabem o que preencher. Placeholder não é suficiente.

Leitores de tela: "campo de texto, campo de texto" (sem contexto)

<input type="text" placeholder="Digite seu nome" />
<!-- PROBLEMA: Sem label associado -->
<!-- Placeholder some ao focar e não é lido como label -->

Versão Corrigida

Solução: Associar label ao input usando for + id ou input dentro do label.

Leitores de tela: "Nome, campo de texto", "Email, campo de texto"

<!-- Opção 1: Label com for + input com id -->
<label for="nome">Nome:</label>
<input type="text" id="nome" name="nome" />

<!-- Opção 2: Input dentro do label -->
<label>
  Nome:
  <input type="text" name="nome" />
</label>

<!-- Opção 3: aria-label (use apenas se label visual não for viável) -->
<input type="text" aria-label="Nome" name="nome" />

Por que funciona: Labels criam associação semântica. Quando um input recebe foco, leitores de tela anunciam o texto do label associado. Isso é essencial para formulários acessíveis.

Playground 6: Uso Incorreto de role e aria-*

Versão com Problema

Problema: Uso incorreto de ARIA (aria-hidden em elemento interativo, role redundante, aria-label inconsistente).

Div como botão

Problemas: botão invisível para leitores, div interativa sem teclado, atributos duplicados

<!-- PROBLEMA: Botão oculto de leitores de tela -->
<button aria-hidden="true">Ação importante</button>

<!-- PROBLEMA: Role redundante (button já é button) -->
<button role="button">Clique</button>

<!-- PROBLEMA: Duplo aria-label -->
<button aria-label="Enviar" aria-label="Submit">...</button>

Versão Corrigida

Solução: Usar ARIA apenas quando necessário, com semântica HTML adequada e atributos consistentes.

Todos acessíveis: sem aria-hidden em interativos, sem roles redundantes, aria-label único

<!-- CORRETO: Use elementos semânticos HTML -->
<button>Ação importante</button>

<!-- CORRETO: Se precisar de elemento customizado, use ARIA corretamente -->
<div role="button" 
     tabindex="0"
     aria-label="Clique para enviar"
     onclick="handleClick()"
     onkeydown="handleKeydown(event)">
  Enviar
</div>

<!-- CORRETO: aria-label único e descritivo -->
<button aria-label="Enviar formulário de contato">Enviar</button>

Regra de ouro ARIA: "Não use ARIA quando HTML semântico funciona". Use <button> em vez de <div role="button">. Não use aria-hidden="true" em elementos interativos. Não duplique atributos.
