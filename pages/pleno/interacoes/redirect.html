Informações para Testes

Cenários de Teste:

  • Clicar em botão de redirecionamento
  • Interceptar redirecionamento em testes automatizados
  • Validar URL após redirecionamento
  • Testar redirecionamentos com delay
  • Validar que redirecionamento não ocorre em modo de teste

Elementos identificáveis:

  • data-testid="redirect-btn" - Botão redirecionar
  • data-testid="redirect-delay-btn" - Botão redirecionar com delay
  • data-testid="redirect-same-page-btn" - Botão redirecionar interno
  • data-testid="redirect-status" - Status do redirecionamento

Exemplos de Teste Automatizado:


// Cypress - Testar redirecionamento (interceptado)
cy.visit('/pages/interacoes/redirect.html')
cy.get('[data-testid="redirect-btn"]').click()
cy.url().should('include', 'redirect.html')
// Em testes reais, você interceptaria:
// cy.intercept('GET', '**/google.com').as('redirect')
// cy.get('[data-testid="redirect-btn"]').click()
// cy.wait('@redirect')
                

// Playwright - Testar redirecionamento
await page.goto('/pages/interacoes/redirect.html')
await page.getByTestId('redirect-btn').click()
// Interceptar redirecionamento
const [response] = await Promise.all([
  page.waitForNavigation(),
  page.getByTestId('redirect-btn').click()
])
// Validar URL
expect(page.url()).toContain('redirect')
                

// Selenium Java - Testar redirecionamento
driver.get(".../pages/interacoes/redirect.html");
String currentUrl = driver.getCurrentUrl();
driver.findElement(By.cssSelector("[data-testid='redirect-btn']")).click();
// Aguardar redirecionamento (se necessário)
Thread.sleep(2000);
String newUrl = driver.getCurrentUrl();
// Validar que URL mudou ou foi interceptada
                

# Capybara (Ruby) - Testar redirecionamento
visit '/pages/interacoes/redirect.html'
find('[data-testid="redirect-btn"]').click
# Validar que redirecionamento foi interceptado ou ocorreu
expect(page).to have_current_path(/redirect/)
