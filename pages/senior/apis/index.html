Sobre a API Mockada

A API mockada foi criada especificamente para prática de testes QA. Ela simula comportamentos reais de backend, incluindo sucessos, erros (4xx, 5xx), autenticação, autorização e regras de negócio.

Autenticação

  • Login/Logout
  • Tokens Bearer
  • Validação de tokens
  • Controle de permissões

Usuários

  • GET /users (admin)
  • POST /users (admin)
  • PUT /users/:id
  • DELETE /users/:id (admin)

Pedidos

  • GET /orders
  • POST /orders
  • PUT /orders/:id
  • DELETE /orders/:id (admin)

Cenários

  • Sucesso (200, 201, 204)
  • Erros 4xx (400, 401, 403, 404, 409, 422, 429)
  • Erros 5xx (500, 503)
  • Timeout (504)

Dica: Use query param ?scenario=<tipo> ou header x-mock-scenario para controlar o comportamento da API em cada requisição. Perfeito para testar diferentes cenários sem modificar código!
